Present this overview to the user, adapting the wording to fit the conversation:

## What Graphit is

Graphit turns questions about your business data into governed, shareable answers. Ask in plain language and Graphit answers either as a fast query over cached data or as a hand-authored, interactive HTML dashboard.

Everything runs through a governed semantic layer - your team's shared definitions of every metric, dimension, and rule - so numbers are computed the team's way, reusable, and safe to share.

## First things to try

- Ask a data question: "how is retention trending this month?", "what's revenue by plan?"
- Build a dashboard: "build me a dashboard for weekly active users and spend"
- Manage your knowledge base: define a metric, a dimension, or a governance rule so future answers reuse it

## Plugin commands

- `/graphit:doctor` - check plugin health: installed vs latest version, login status, and anything needing attention
- `/graphit:update` - update the plugin and CLI to the latest version
- `/graphit:login` - log in to Graphit (opens your browser)
- `/graphit:help` - this overview

Keep it concise. If the user already seems oriented, offer the first-things-to-try list and stop.
